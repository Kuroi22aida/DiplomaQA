# План автоматизации проверки покупки тура

## Перечень автоматизируемых сценариев

### Позитивные проверки

**_Кейс 1. Покупка тура обычной оплатой дебетовой картой_**

Нажать клавишу *"Купить"*. Ввести номер активной дебетовой карты - *4444 4444 4444 4441*. Заполнить поле **месяц** значением текущего месяца. Заполнить поле **год** двумя последними цифрами текущего года либо со сдвигом вперед на срок до 5 лет. Ввести **имя** и **фамилию** пользователя 
на латиннице. Ввести 3 цифры **CVC**-кода. Нажать кнопку
*"Оплатить"*.

**Ожидаемый результат:** Сообщение об успешной транзакции. В БД приложения сохранена информация об успешной покупке.

**_Кейс 2. Покупка тура в кредит по данным банковской карты_**

*"Оплатить"*.

**Ожидаемый результат:** Сообщение об одобренной заявке на кредит. В БД приложения сохранена информация об успешной покупке.

**_Кейс 3. Граничный месяц выпуска карты в форме покупки тура_**

## Перечень используемых инструментов с обоснованием выбора

- IntelliJ IDEA 2022.1 (Community Edition) - самая популярная IDE для JAVA
- VM: OpenJDK 64-Bit Server VM by JetBrains s.r.o. - бесплатное ПО для создания приложений Java
- СУБД MySQL, PostgreSQL - согласно ТЗ
- Docker Desktop, Docker Compose - для запуска СУБД и Node.js (для симуляции банковсих сервисов). Бесплатное ПО для виртуализации приложений
- Gradle - для автоматизации сборки и запуска тестов, один из популярных инструментов
- JUnit 5 - наиболее широко используемая среда тестирования для приложений Java
- Selenide - удобный фреймворк для автоматизированного тестирования веб-приложений на основе Selenium WebDriver
- Faker - удобный инструмент для генерации тестовых данных
- Allure - информативный, наглядный, легкий в установке инструмент для генерации отчетов
- DBUtils - для упрощения кода работы с БД
